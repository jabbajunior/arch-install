# Purpose
The purpose of this project is to develop a deeper understanding of the underlying systems that power Linux by performing a manual Arch Linux installation and documenting the full process.

While I currently have a solid foundation in user-space tools (completed Parts 1-3 of the *Linux Command Line* by William Shotts), this project focuses on exploring lower-level system components such as filesystems, systemd services, boot processes, storage configuration, and networking.
# Learning Objectives
- Gain hands-on experience installing and configuring a Linux system from scratch
- Understand the tradeoffs between key system design decisions (e.g., filesystem selection, boot configuration, networking setup)
- Being able to reproduce a full system deployment using installations scripts
# Project Scope
This project consists of a single, isolated Arch Linux installation environment intended purely for learning and experimentation. The focus is on understanding the installation and system configuration process rather than building a production-ready workstation.

To keep the scope manageable, I am working with these constraints:
- Filesystem encryption is excluded
- The primary objective is a working base system installation, not end-user desktop configuration
- Decisions will prioritize learning clarity not advanced optimization
# Long-Term Outcome
The long-term objective of this project is to develop a deeper understanding of the Linux operating system by building, configuring, and rebuilding an Arch Linux installation from scratch. Through documenting decisions, researching system design tradeoffs, and eventually creating scripts capable of reproducing the full installation, this project aims to strengthen practical knowledge of Linux internals and system administration fundamentals while establishing a reproducible system deployment workflow.
